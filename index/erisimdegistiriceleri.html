Erişim Değiştiriciler

Şimdiye kadar, görünen anahtar kelimeye oldukça aşinasınız. örneklerimizin çoğunda:public

public string color;

Anahtar kelime bir erişim değiştiricidir, sınıflar, alanlar, yöntemler ve özellikler için erişim düzeyini/görünürlüğü ayarlamak
için kullanılır.public

Özel Değiştirici

Erişim değiştiricisi olan bir alan bildirirseniz, yalnızca aynı sınıf içinde erişilenler:private

Örnek

class Car
{
private string model = "Mustang";

static void Main(string[] args)
{
Car myObj = new Car();
Console.WriteLine(myObj.model);
}
}

Çıktı şöyle olacaktır:

Mustang

Sınıf dışından erişmeye çalışırsanız, bir hata oluşur:

Örnek

class Car
{
private string model = "Mustang";
}

class Program
{
static void Main(string[] args)
{
Car myObj = new Car();
Console.WriteLine(myObj.model);
}
}

Çıktı şöyle olacaktır:

'Car.model' is inaccessible due to its protection level
The field 'Car.model' is assigned but its value is never used

Genel Değiştirici

Erişim değiştiricisi olan bir alan bildirirseniz, tüm sınıflar:public

Örnek

class Car
{
public string model = "Mustang";
}

class Program
{
static void Main(string[] args)
{
Car myObj = new Car();
Console.WriteLine(myObj.model);
}
}

Çıktı şöyle olacaktır:

Mustang

Neden Erişim Değiştiriciler?
Sınıf üyelerinin görünürlüğünü denetlemek için ( güvenlik düzeyi her bir sınıf ve sınıf üyesi).

"Kapsülleme" yi başarmak için - bu, "hassas" verilerin kullanıcılardan gizlendiğinden emin olma işlemidir. Bu, alanları . Bir sonraki bölümde bu konuda daha fazla bilgi edineceksiniz.private

Not: Varsayılan olarak, bir sınıfın tüm üyeleri bir erişim değiştiricisi belirtmezseniz:private

Örnek

class Car
{
string model; // private
string year; // private
}
