Özellikler ve Kapsülleme

Özellikleri açıklamaya başlamadan önce, "Kapsülleme" hakkında temel bir anlayışa sahip olmalısınız.

Kapsüllemenin anlamı, "hassas" verilerin gizlendiğinden emin olmaktır. kullanıcılardan. Bunu başarmak için şunları
yapmalısınız:

  • alanları/değişkenleri şu şekilde bildir private
  • özellikler aracılığıyla, bir alanın değerine erişmek ve bunları güncelleştirmek için yöntemler sağlar ve yöntemler public/get/set/private

Özellikler

Bir önceki bölümden değişkenlerin sadece aynı sınıf içinde erişilir (bir dış sınıfın buna erişimi yoktur). Fakat bazen onlara
erişmemiz gerekir - ve bu özelliklerle yapılabilir.private

Bir özellik, bir değişken ve bir yöntemin birleşimi gibidir ve iki yöntemi vardır: a ve a yöntemi:getset

Örnek

class Person
{
private string name; // field

public string Name // property
{
get { return name; } // get method
set { name = value; } // set method
}
}

Örnek açıklandı

Özellik, alanla ilişkilendirilir. Hem özellik hem de özel alan için aynı adı, ancak ilk harfi büyük olacak şekilde kullanmak
iyi bir uygulamadır.Name/name

Yöntem, değişkenin değerini döndürür.get/name

Yöntem, değişkene a atar. Anahtar kelime, mülke atadığımız değeri temsil eder.set/value/name/value

Tam olarak anlamadıysanız, aşağıdaki örneğe bir göz atın.

Artık sınıfın alanına erişmek ve güncellemek için özelliği kullanabiliriz:Name/private/Person

Örnek

class Person
{
private string name; // field
public string Name // property
{
get { return name; }
set { name = value; }
}
}

class Program
{
static void Main(string[] args)
{
Person myObj = new Person();
myObj.Name = "Liam";
Console.WriteLine(myObj.Name);
}
}

Çıktı şöyle olacaktır:

Liam

Otomatik Özellikler (Kısa El)

C# ayrıca, kısa el / otomatik özellikleri kullanmak için bir yol sağlar. özellik için alanı tanımlamanız gerekmez ve
yalnızca mülkün içine ve içine yazmanız gerekir.get;set;

Aşağıdaki örnek, yukarıdaki örnekle aynı sonucu verecektir. Tek fark, daha az kod olmasıdır:

Örnek

Otomatik özellikleri kullanma:

class Person
{
public string Name // property
{ get; set; }
}

class Program
{
static void Main(string[] args)
{
Person myObj = new Person();
myObj.Name = "Liam";
Console.WriteLine(myObj.Name);
}
}

Çıktı şöyle olacaktır:

Liam

Neden Enkapsülasyon?

  • Sınıf üyelerinin daha iyi kontrolü (kendinizin (veya başkalarının) kodu bozma olasılığını azaltın)
  • Alanlar salt okunur (yalnızca yöntemi kullanıyorsanız) veya salt yazma (yalnızca yöntemi kullanıyorsanız)
    yapılabilir get/set
  • Esnek: programcı, diğer parçaları etkilemeden kodun bir bölümünü değiştirebilir
  • Artan veri güvenliği
